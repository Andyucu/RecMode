1. Overview

RecMode is a modern Windows screen recorder built with .NET 10 and WPF. It is designed for practical desktop recording: tutorials, bug reports, meetings, gameplay, screenshots, short clips, scheduled recordings, and repeatable recording profiles.

The app is portable-first. The portable package can run from a folder or USB drive and keeps its data beside the app. The installer package uses Velopack and includes the same local runtime assets, including the bundled ffmpeg tools used for recording and media probing.

Core idea: choose a source, choose a profile or encoding settings, decide what audio to include, then press Record. RecMode handles capture, encoding, library indexing, and optional remux/recovery behavior. 
Typical recording path

Source        GPU processing        Encoder process        Library
Screen ----\  crop/scale/NV12  ---> ffmpeg.exe -------> video file + metadata
Window ----+  cursor/webcam         H.264/HEVC/AV1       Library entry
Region ----/  click/ink overlay     AAC/Opus/FLAC        Screenshot/video list

3. Installation Options

Portable Zip

Use the portable zip when you want RecMode to be self-contained in one folder.

  1. Extract the zip to a folder such as D:\Apps\RecMode.
  2. Run RecMode.exe.
  3. Recordings are stored in Recordings\ by default.
  4. Settings, logs, and the library index are stored in Data\.

Setup Installer

RecMode-win-Setup.exe is a one-click Velopack installer. By default it installs per-user under LocalAppData and creates shortcuts.

To override the install folder from a command line:

RecMode-win-Setup.exe --installto "D:\Apps\RecMode"

MSI Installer

RecMode-win.msi is provided for users or administrators who prefer Windows Installer deployment. It is generated with install-location mode set to Either, so it supports per-user/per-machine scope selection in the MSI flow.

Administrators can override the MSI target directory with:

msiexec /i RecMode-win.msi VELOPACK_INSTALLDIR="D:\Apps\RecMode"

4. Quick Start

  1. Open RecMode.
  2. On the Record page, choose a source: Screen, Window, or Region.
  3. Choose a recording profile or set video options manually.
  4. Choose audio sources: system audio, microphone, or a specific app audio target.
  5. Optionally enable webcam picture-in-picture.
  6. Press Record or use F9.
  7. Use Pause/Resume with F10 if needed.
  8. Stop the recording with the Stop button or F9.
  9. Find the video in the Library page or in the output folder.

5. Feature Reference

Capture Sources

SourceUse it forNotes
ScreenRecording a whole display.On multi-monitor systems, RecMode can also expose an All Displays option.
WindowRecording one application window.The Follow selected window toggle lets RecMode re-resolve a selected app window if the app recreates its HWND. Normal movement and resizing are already followed by Windows Graphics Capture.
RegionRecording a rectangular area.Use Change to pick a new region. Region coordinates are stored and reused.
Webcam overlayAdding camera picture-in-picture on top of another source.Webcam is an overlay, not currently a standalone primary source.

Recording Profiles

Profiles bundle common recording choices: container, frame rate, quality, and audio settings. They intentionally do not choose the encoder, because hardware encoder availability depends on the machine.

Built-in profileTypical use
Tutorial30 fps MP4 with system audio and microphone.
Gameplay60 fps MP4 with higher quality and system audio.
Meeting30 fps MP4, smaller file size, system audio and microphone.
Bug report30 fps MP4, moderate quality, simple shareable output.
GIF clip15 fps short silent clip settings. This is not true GIF export.
High-quality archiveMKV, high quality, FLAC audio.

Use Save as to store your current settings as a custom profile. Use the new Next profile hotkey, F8 by default, to cycle through built-in and custom profiles without opening the profile combo. The hotkey skips Custom and is ignored while recording.

Encoding and Formats

  • Codecs: H.264, HEVC/H.265, and AV1 are modeled in the UI and selected based on encoder availability.
  • Backends: Auto, NVENC, AMF, QSV, and software paths are probed rather than assumed.
  • Containers: MP4, MKV, MOV, and WebM.
  • Frame rates: common choices include 15, 30, 60, and 120 fps.
  • Quality: the slider maps to ffmpeg quality settings. Higher quality means larger files.
  • Safe recording: RecMode favors crash-resilient capture behavior where possible; MKV is more recoverable than MP4 if recording is interrupted.

Audio

  • System audio toggle.
  • Microphone toggle.
  • Per-source volume sliders and level meters.
  • Optional Limit to app selector for per-app audio capture.
  • Audio formats include AAC, Opus, and FLAC depending on container compatibility.

Screenshots, Library, and Record Again

  • Press Screenshot or F11 to capture the current source as an image.
  • The Library page lists videos and screenshots.
  • Open, Show in folder, Delete, Refresh, and Open folder are available from the library.
  • Record again applies a previous recording's container, frame rate, quality, and audio toggles back to the Record page.

Scheduling

The Schedule page can start recordings automatically while RecMode runs. A schedule can follow current Record settings or bind to a recording profile. Source and encoder follow current Record settings; frame rate, quality, container, and audio can come from the schedule's selected profile.

Overlays and Capture Helpers

  • Countdown overlay before interactive recording starts.
  • Floating recording toolbar with record state controls.
  • Click ripple overlay for mouse click emphasis.
  • Draw-on-screen annotation overlay during recording.
  • Webcam picture-in-picture overlay with position and size controls.

6. Common Workflows

Record a Tutorial

  1. Select Screen or Region.
  2. Pick the Tutorial profile.
  3. Enable microphone and system audio.
  4. Enable click highlights if you want visible click feedback.
  5. Press F9, wait for the countdown, then record.

Record a Single App Window

  1. Select Window source.
  2. Choose the app window from the picker.
  3. Leave Follow selected window enabled if the app may recreate its window.
  4. Choose audio settings. Use Limit to app if you only want that app's audio.
  5. Start recording.

Capture a Bug Report

  1. Select Region or Window.
  2. Choose the Bug report profile.
  3. Use the draw overlay or click highlights to point out the issue.
  4. Stop recording and use Library > Show in folder to attach the file.

Create a Scheduled Recording

  1. Set the Record page source and encoder as you want them.
  2. Create or choose a profile for the recording quality/audio settings.
  3. Open Schedule and create a new schedule.
  4. Choose recurrence, start time, duration, and optional profile.
  5. Keep RecMode running, including in the tray.

ItemLocation

Privacy posture: RecMode is designed as a local desktop tool. Crash minidumps are opt-in. Update checking is configurable. The project notes explicitly treat privacy as a product feature.

9. Troubleshooting

ProblemWhat to try
No hardware encoder appearsUse Auto or Software. Hardware encoders depend on GPU vendor, driver, and ffmpeg support. RecMode probes actual encoder startup rather than trusting ffmpeg's compiled list.
Recording is choppy or encoder cannot keep upLower frame rate, lower quality, use hardware encoding, choose Faster effort, reduce capture size, or enable lower encoder process priority.
Output file is too largeLower quality, use 30 fps, use AV1/HEVC where supported, or choose a profile like Meeting or Bug report.
Window source no longer captures the intended appTurn on Follow selected window and reselect the window once. If several windows from the same process exist, choose the exact one again.
Microphone or webcam is unavailableCheck Windows privacy permissions, device connection, and whether another app is exclusively using the device.
Installer does not ask for a folderUse the MSI for an interactive Windows Installer flow, or run Setup.exe with --installto.

10. Beta Notes and Known Limits

  • This document covers 0.9.19-beta. The version is intentionally pre-1.0 while verification continues.
  • Full-system audio has a known beta issue in the project memory: recordings may contain a valid but silent stream. Per-app audio targeting has been verified separately and is the safer audio path until the full-system issue is fixed.
  • NVENC and QSV still need vendor-hardware verification on real NVIDIA and Intel systems. Development hardware has been AMD.
  • Real webcam hardware verification, real microphone hardware verification, a Narrator screen-reader pass, and Windows 10 2004 regression testing are still tracked as standing validation gaps.
  • True GIF export, editing tools, and advanced library/share workflows are not covered by the current beta feature set.
