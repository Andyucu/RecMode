### Added
- 2026-07-04 — **Phase 0.5 (Pipeline spike) complete — ⚠️ decision gate PASSED (Tier 1).** Disposable spike (`tools/spike/`) proved the full spine end-to-end: WGC monitor capture → D3D11 → GPU BGRA→NV12 (VideoProcessor, single pass) → staging readback → named pipe → ffmpeg → file, CFR-paced, plus a second WASAPI system-audio loopback pipe muxed to AAC. **Validated on AMD (RX 7900 XTX / Ryzen 7 7700X), so the hardware path measured is `h264_amf`** (not NVENC — the plan's dev-machine assumption). **5-minute 1440p60 h264_amf run: 17,992/18,000 CFR frames (0 steady-state drops), 316 MB/s sustained pipe throughput, app CPU 10.2% of one core excluding ffmpeg, memory flat (~108 MB) — all three gate criteria pass.** Encoder/container matrix (h264_amf + libx264 × MP4 + MKV) all produce ffprobe-valid files; native 5120×1440 via libx264 sustained 620 MB/s with 0 stalls (pipeline headroom). Kill-ffmpeg-mid-recording test confirmed the safe-recording design: MP4 dies ("moov atom not found") while a truncated MKV remuxes (`-c copy`) into a clean playable MP4 — recovery proven. Findings: h264_amf caps at 4096 width (ultrawide needs HEVC/AV1 or downscale); a one-time ~95 ms AMF encoder-init stall at t=0; ffmpeg death surfaces as a broken-pipe `IOException` that production must handle (→ `FatalFinalizationError`). Staged the pinned BtbN GPL ffmpeg build in `tools/ffmpeg/` with a committed SHA-256 `ffmpeg.manifest.json`. Added Vortice.Direct3D11/DXGI + NAudio to central package management. **Verdict: no Tier 2 (in-proc hw frames) needed for MVP; NVENC/QSV need their own gate re-check on NVIDIA/Intel hardware.**
- 2026-07-04 — **Phase 0 (Foundation & scaffolding) complete.** First application code lands. `git init`; solution (`RecMode.slnx`) with 6 src + 3 test projects; `Directory.Build.props` (net10.0-windows10.0.19041.0, nullable, x64, analyzers) + central package management. `RecMode.Core`: error taxonomy (4 severities + reporter, §3.6), recording state machine with pause-PTS/gapless math and monotonic-clock abstraction (§3.7), versioned JSON settings service (atomic writes, corrupt-file recovery, schema migration), `AppPaths` portable/installed path chokepoint (§3.5), `OsCapabilities` single OS-version gate, crash reporter (session sentinel + crash log). `RecMode.Interop`: `MiniDumpWriteDump` writer (opt-in). `RecMode.Encoding`: `FfmpegLocator` (bundled/user-override resolution + SHA-256 manifest verification, §3.4). `RecMode.App`: WPF host with Microsoft.Extensions.Hosting DI, Serilog file logging, global crash handlers, PerMonitorV2 manifest, DI'd shell window showing foundation health. `build.ps1` + `publish-portable.ps1` (self-contained ReadyToRun win-x64 zip) + `portable.marker` + ffmpeg/licenses scaffolding. Build clean (0 warnings); 33 unit tests pass; app verified launching in portable mode. Added `PROJECT_MEMORY.md` engineering build log.
